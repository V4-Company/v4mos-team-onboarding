Arquitetura Interna

Clean Architecture + DDD

O v4-marketing-backend segue camadas estritas com dependency inversion

Domain
Aggregates Value Objects Repository Interfaces Domain Events
Zero dependencias externas
depende de
App
Use Cases Services DTOs (Zod) Event Handlers
Depende apenas do Domain
depende de
Infra
Controllers (Fastify) Repositories (Kysely) DI Container (Inversify) Providers (Axios, AWS)
Depende de Domain + App
O servico chat usa apenas 2 camadas (app + infra) por nao ter entidades de dominio complexas. Regras enforced via eslint-ddd-plugin.mjs.
Setup

Quick Start

Do zero ao ambiente rodando em minutos

1

Clone o onboarding

terminal 
git clone git@github.com:V4-Company/v4mos-team-onboarding.git
cd v4mos-team-onboarding
2

Rode o bootstrap

Clona os 3 repos, instala deps e cria os .env

terminal 
./scripts/bootstrap.sh
3

Configure o .env

Peca as variaveis sensiveis (DB, AWS, Clerk) para alguem do time via 1Password ou DM.

4

Suba a infra e rode

terminal — v4-marketing-backend 
# Sobe Postgres, LocalStack, RabbitMQ + roda migrations
pnpm dev:prepare

# Roda todos os servicos + frontend
pnpm dev:all
5

Abra no VS Code

terminal 
code v4mos.code-workspace

Abre os 3 repos como multi-root workspace com extensoes recomendadas.

Referencia

Comandos Essenciais

Cheat sheet dos comandos que voce vai usar todo dia

v4-marketing-backend

pnpm install Instalar dependencias
pnpm build:lib Build da lib compartilhada (rode primeiro!)
pnpm dev:prepare Sync env + sobe infra Docker + migrations
pnpm dev:all Roda todos os servicos + frontend
pnpm dev:backend Roda apenas os servicos backend
pnpm test Rodar testes (Vitest)
pnpm lint ESLint + DDD rules
pnpm typecheck Type checking
cd packages/<svc> && pnpm migrate Rodar migrations Prisma

v4-marketing-api

npm install Instalar dependencias
npm run start:dev Dev server com hot reload
npm test Testes unitarios (Jest)
npm run test:e2e Testes end-to-end
npm run lint ESLint

v4-marketing-data

yarn install Deps Node (SST)
uv sync Deps Python
npx sst dev Dev mode (requer AWS creds)
npx sst deploy Deploy para AWS
Onboarding

Sua Primeira Semana

Roadmap dia a dia pra voce se sentir em casa

Dia 1

Setup & Contexto

  • Rodar o bootstrap e ter tudo funcionando
  • Ler a doc de arquitetura
  • Acessar o frontend em localhost:3000
  • Pedir acessos: GitHub, Clerk, AWS, Datadog, Slack
Dia 2

Explorar o Monorepo

  • Ler o CLAUDE.md (doc tecnica completa)
  • Navegar domain/app/infra do v4-marketing
  • Entender o fluxo: Controller → UseCase → Repository
  • Rodar os testes: pnpm test
Dia 3

DDD & Convencoes

  • Entender a separacao de camadas e as regras DDD
  • Olhar o DI container (Inversify)
  • Entender Prisma (migrations) + Kysely (queries)
  • Criar uma branch e fazer uma mudanca trivial
Dia 4

Outros Repos

  • Explorar v4-marketing-api (modulos de integracoes)
  • Explorar v4-marketing-data (pipelines SST)
  • Entender como os repos se conectam
Dia 5

Primeiro PR

  • Pegar uma task pequena do backlog
  • Abrir PR seguindo as convencoes
  • Code review com alguem do time
  • Pair programming com um senior
