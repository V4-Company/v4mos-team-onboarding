Arquitetura Interna

Clean Architecture + DDD

O v4-marketing-backend segue camadas estritas com dependency inversion

Domain
Aggregates Value Objects Repository Interfaces Domain Events
Zero dependencias externas
depende de
App
Use Cases Services DTOs (Zod) Event Handlers
Depende apenas do Domain
depende de
Infra
Controllers (Fastify) Repositories (Kysely) DI Container (Inversify) Providers (Axios, AWS)
Depende de Domain + App
O servico chat usa apenas 2 camadas (app + infra) por nao ter entidades de dominio complexas. Regras enforced via eslint-ddd-plugin.mjs.

authorizer

Lambda AWS que valida tokens JWT e acesso a workspaces no API Gateway. Suporta user tokens e service accounts.

AWS Lambda Inversify Kysely

Fluxo de Validacao

1. Recebe request do API Gateway
2. Detecta metodo: Authorization header ou x-client-id
3a. User token: valida via Identity service + checa workspace membership
3b. Service account: valida x-client-id + x-client-secret
4. Retorna IAM policy (Allow/Deny)

Estrutura

Setup

Quick Start

Do zero ao ambiente rodando em minutos

1

Clone o onboarding

terminal 
git clone git@github.com:V4-Company/v4mos-team-onboarding.git
cd v4mos-team-onboarding
2

Rode o bootstrap

Clona os 3 repos, instala deps e cria os .env

terminal 
./scripts/bootstrap.sh
3

Configure o .env

Peca as variaveis sensiveis (DB, AWS, Clerk) para alguem do time via 1Password ou DM.

4

Suba a infra e rode

terminal — v4-marketing-backend 
# Sobe Postgres, LocalStack, RabbitMQ + roda migrations
pnpm dev:prepare

# Roda todos os servicos + frontend
pnpm dev:all
5

Abra no VS Code

terminal 
code v4mos.code-workspace

Abre os 3 repos como multi-root workspace com extensoes recomendadas.
